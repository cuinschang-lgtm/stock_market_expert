Project Insight · 02 / 09
项目动机

投资研究需要更好的工具， 但不是另一个"喊单机器人"

真正的需求：数据整理、研究分析、逻辑追踪、事件提醒

市面上的股票工具要么只有行情展示，要么直接给买卖建议。我们想做的是一个真正帮助投资者完成投研闭环的工作台——整理数据、沉淀逻辑、追踪事件。

我们的原则：不喊单，不输出买卖建议，不传递内幕信息。所有分析仅用于信息整理与辅助研究。 核心原则 · 从产品第一天起就确定的边界
Pain Point 01
数据分散在不同平台。
行情、财报、公告、行业新闻分散在多个 App，缺乏统一的投研视图。
Pain Point 02
投资逻辑难以沉淀。
"为什么会关注这只股票"的判断往往停留在脑子里，没有结构化的记录和追踪。
Our Answer
一个可持续迭代的全栈投研工作台。
覆盖 A 股、港股、美股，从搜索自选到生成分析再到保存笔记，形成完整闭环。
为什么做？因为投研工具不该是喊单机器人
真正的价值在于帮助用户建立自己的研究体系
Architecture · Full-Stack Design
Tech Stack · 03 / 09
技术架构

Next.js 14 App Router + 三层数据抽象

前端统一通过 MarketDataProvider 获取数据，页面层零依赖具体数据源。支持 mock / Supabase 云端 / 外部 API 三种模式无缝切换。

01
表现层
React 18 Server Components + Tailwind CSS + Recharts 图表。26 条路由全部 Server-Side Rendering，首屏快速加载。
02
API 层
11 个 Next.js API Routes，覆盖搜索/行情/K线/财务/事件/分析/笔记 CRUD。统一 JSON 响应格式 + 错误兜底。
03
数据层
MarketDataProvider 接口抽象。Mock 模式零依赖即可运行；配置 Supabase 后自动切换云端持久化。
04
AI 编排层
结构化投研分析引擎。生成包含位置/估值/事件/风险四模块的标准化报告，所有结论标注数据来源和时间戳。
05
持久化层
Supabase PostgreSQL + 浏览器 localStorage 双层降级。未配置云端时自动回退本地存储，用户数据不丢失。
06
部署层
Vercel 一键部署（香港节点），零配置即可上线。vercel.json 指定 hkg1 区域，改善中国大陆访问体验。
前端 → API → Provider → 数据源 · 每一层都独立可替换
架构的设计质量决定了项目的可维护性

Data Layer · Provider Pattern
Architecture · 05 / 09
数据流设计

MarketDataProvider 抽象层
让数据源可替换、页面零改动

这是整个项目最核心的架构决策。所有页面组件不直接依赖任何具体数据源，而是通过统一的 TypeScript 接口获取数据。

Step 1
搜索 & 行情
searchSymbols / getQuote / getKline —— 支持 A股港股美股的统一行情接口
Step 2
财务 & 事件
getFinancials / getCompanyEvents —— 财报、公告、行业事件的标准化结构
Step 3
行业 & 仪表盘
getSectorOverview / getDashboardData —— 板块数据和市场总览
Step 4
Provider 切换
Mock → Qclaw → External，环境变量一键切换，页面代码零修改
当前 Mock Provider 覆盖 5 只示例股票（茅台/腾讯/NVIDIA/宁德/Apple），含完整行情、K线、财务和事件数据，可独立跑通全部功能。 接入真实数据源时只需新增 Provider 实现，页面层完全不受影响
接口抽象是可持续迭代的基础 · 开发期 mock，上线期切换真实数据
好的架构让未来替换成本趋近于零
AI Analysis · Structured Research
Intelligence · 06 / 09
AI 投研编排

结构化分析 · 不是聊天框，是研究报告

AI 分析模块不是简单的 Q&A 聊天，而是按照"当前位置→估值水位→短期事件→风险提示"的标准模板，生成可追溯、可保存的结构化报告。

AI Analyst · 结构化投研分析 四模块标准化报告 / 可保存为笔记
行业主题研究 · 7 大板块 龙头样本 / 跟踪变量 / 选股线索
每份分析报告必含数据来源、时间戳和免责声明
不输出买卖建议，只帮助用户理清变量和风险

Conclusion · 08 / 09
项目总结

一个真正"能用"的全栈项目

① 架构先行：MarketDataProvider 抽象层让数据源可替换、页面零改动，这是项目可持续演进的基石。
② 功能完整：9 个页面覆盖了从搜索到分析到报告导出的完整投研闭环，远超 MVP 目标。
③ 工程质量：TypeScript strict 零错误、全量类型定义、Client/Server Component 分离、双层持久化降级。
从"搭一个能跑的网站"到"设计一个能迭代的系统"——这才是全栈开发真正的挑战。
